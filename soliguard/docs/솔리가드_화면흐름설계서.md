# 솔리가드(SoliGuard) 화면 흐름 설계서

## 전체 화면 흐름도

사용자의 핵심 동선은 다음과 같이 흐릅니다.

- **최초 실행 시**: 직무 선택(온보딩) → 메인 대시보드
- **일상 사용 시**: 메인 대시보드 → 스캔 설정 → 스캔 진행 → 검출 결과 → 조치(마스킹/격리/삭제) → 완료 및 리포트
- **보조 흐름**: 대시보드 → 격리함 / 이력·감사 로그 / 설정

```
[온보딩: 직무 선택] (최초 1회)
        │
        ▼
[메인 대시보드] ──┬──→ [스캔 설정] → [스캔 진행] → [검출 결과] → [조치 확인] → [완료/리포트]
                  ├──→ [격리함]
                  ├──→ [이력/감사 로그]
                  └──→ [설정]
```

---

## 화면 1. 온보딩 — 직무 선택 (최초 실행 시 1회)

이 화면의 목적은 사용자의 직무를 받아 역할별 스캔 프로파일을 자동 구성하는 것입니다. SI 특화의 핵심 차별점이 첫 화면에서 드러납니다.

```
┌─────────────────────────────────────────────┐
│              🛡 SoliGuard                     │
│       내 PC의 고객 데이터, 먼저 찾습니다       │
│                                               │
│   어떤 업무를 하시나요?                        │
│   (선택한 직무에 맞춰 점검 항목을 구성합니다)   │
│                                               │
│   ┌────────┐ ┌────────┐ ┌────────┐          │
│   │ 개발자  │ │디자이너 │ │ 기획자  │          │
│   └────────┘ └────────┘ └────────┘          │
│   ┌────────┐ ┌────────┐                      │
│   │  PM    │ │전산사무 │                      │
│   └────────┘ └────────┘                      │
│                                               │
│   ☐ 모든 데이터는 이 PC 안에서만 처리됩니다     │
│                                               │
│                          [ 시작하기 ]         │
└─────────────────────────────────────────────┘
```

직무 카드를 선택하면 테두리가 강조됩니다. 하단의 "모든 데이터는 이 PC 안에서만 처리됩니다" 문구는 신뢰 설계의 핵심이므로 첫 화면부터 노출합니다. 시작하기를 누르면 메인 대시보드로 이동합니다. 직무는 나중에 설정에서 변경할 수 있습니다.

---

## 화면 2. 메인 대시보드 (홈)

이 화면은 사용자가 매번 처음 보는 허브입니다. 좌측에 내비게이션, 중앙에 현재 PC의 보안 상태 요약과 큰 스캔 버튼, 우측 또는 하단에 최근 활동을 배치합니다.

```
┌──────────────┬──────────────────────────────────────┐
│ 🛡 SoliGuard │   내 PC 개인정보 위험 등급            │
│              │                                       │
│ ▶ 홈         │        ┌───────────────┐             │
│   격리함      │        │   주의  🟡     │  마지막 점검 │
│   점검 이력   │        │  위험 12건     │  3일 전      │
│   설정       │        └───────────────┘             │
│              │                                       │
│  [직무:개발자]│   ┌─────────────────────────────┐    │
│              │   │      🔍  지금 점검하기        │    │
│              │   └─────────────────────────────┘    │
│              │     (전체 스캔)   (빠른 스캔)         │
│              │                                       │
│              │   최근 활동                            │
│              │   · 6/3 스캔 — 위험 12건 발견          │
│              │   · 6/1 격리 — 3개 파일               │
│              │   · 5/28 프로젝트 클로징 점검 완료      │
└──────────────┴──────────────────────────────────────┘
```

중앙의 위험 등급은 안전(초록)/주의(노랑)/위험(빨강) 3단계로 색과 함께 큼직하게 보여 직관적으로 상태를 파악하게 합니다. 가장 크고 눈에 띄는 요소는 "지금 점검하기" 버튼입니다. 빠른 스캔은 직무 프로파일이 지정한 위험 폴더(다운로드, 바탕화면, 작업 폴더)만 빠르게 훑고, 전체 스캔은 드라이브 전체를 검사합니다.

---

## 화면 3. 스캔 설정

스캔 대상과 범위를 정하는 화면입니다. 직무 프로파일에 따라 기본값이 미리 채워져 있어, 사용자는 대부분 그대로 시작 버튼만 누르면 됩니다.

```
┌─────────────────────────────────────────────┐
│  ← 스캔 설정                                  │
│                                               │
│  스캔 대상 폴더                                │
│  ┌─────────────────────────────┐ [폴더 추가] │
│  │ ☑ C:\Users\me\Downloads      │            │
│  │ ☑ C:\Users\me\Desktop        │            │
│  │ ☑ C:\Projects\고객사A         │            │
│  │ ☐ 전체 드라이브 (C:)          │            │
│  └─────────────────────────────┘            │
│                                               │
│  검출할 항목  (직무: 개발자 기본값)            │
│  ☑ 주민등록번호  ☑ 신용카드  ☑ 전화/이메일    │
│  ☑ 계좌번호      ☑ 사업자번호                 │
│  ☑ DB접속정보·API키·비밀번호  ← 개발자 특화    │
│  ☐ 이미지 속 정보(OCR)  ☐ 한글(hwp) 문서      │
│                                               │
│  파일 형식: txt, csv, xlsx, docx, hwp, pdf... │
│                                               │
│              [ 프로젝트 클로징 점검 ]  [ 스캔 시작 ]│
└─────────────────────────────────────────────┘
```

검출 항목의 기본 체크 상태는 직무에 따라 달라집니다. 개발자는 "DB접속정보·API키"가 기본 체크되고, 디자이너는 "이미지 속 정보(OCR)"가 기본 체크됩니다. "프로젝트 클로징 점검" 버튼을 누르면 특정 프로젝트 폴더를 지정해 잔여 고객 데이터를 한 번에 점검·정리하는 전용 모드로 들어갑니다.

## 화면 4. 스캔 진행

스캔이 도는 동안 사용자를 안심시키고 진행 상황을 보여줍니다. OCR이나 전체 스캔은 시간이 걸리므로 진행률과 실시간 발견 건수를 함께 노출합니다.

```
┌─────────────────────────────────────────────┐
│  스캔 진행 중...                               │
│                                               │
│  ▓▓▓▓▓▓▓▓▓▓▓▓▓░░░░░░░  62%                    │
│                                               │
│  검사 중: C:\Projects\고객사A\고객명단.xlsx    │
│  검사한 파일  1,284 / 2,050                    │
│                                               │
│  지금까지 발견                                 │
│   주민등록번호  8     신용카드     2           │
│   API 키        3     전화/이메일  21          │
│                                               │
│                          [ 일시정지 ]  [ 중지 ]│
└─────────────────────────────────────────────┘
```

진행 중에도 발견 건수를 실시간으로 보여주면 사용자가 도구가 일하고 있음을 체감합니다. 중지하면 그때까지 검출된 결과로 결과 화면으로 넘어갑니다.

---

## 화면 5. 검출 결과 (가장 중요한 화면)

검출된 항목을 위험도별로 분류해 보여주고, 사용자가 조치를 결정하는 핵심 화면입니다. 좌측에 필터, 중앙에 결과 목록, 선택 시 우측에 미리보기(마스킹된 형태)를 배치합니다.

```
┌─────────────────────────────────────────────────────────┐
│  점검 결과 — 위험 12건 발견          [전체 선택] [리포트]  │
│                                                           │
│  위험도   파일/위치              검출 항목      [조치 ▼]  │
│  ────────────────────────────────────────────────────   │
│ 🔴높음 ☑ 고객명단.xlsx          주민번호 8건   [격리][삭제]│
│ 🔴높음 ☑ config.yml            API키·DB접속    [마스킹]   │
│ 🟡중간 ☐ 회의록_0312.hwp        전화·이메일 5건 [마스킹]   │
│ 🟡중간 ☐ 시안_신분증.png        주민번호(이미지)[격리]    │
│ 🟢낮음 ☐ test_log.txt          이메일 3건      [마스킹]   │
│  ...                                                      │
│  ────────────────────────────────────────────────────   │
│  선택한 3건에 대해:  [ 마스킹 ]  [ 격리 ]  [ 완전삭제 ]    │
│                                                           │
│  💡 미리보기: 고객명단.xlsx                                │
│     홍길동 / 90****-2****** / 010-****-1234               │
└─────────────────────────────────────────────────────────┘
```

위험도는 색상 아이콘으로 즉시 구분되고, 높은 위험부터 정렬됩니다. 각 행에서 개별 조치도 가능하고, 여러 건을 체크해 하단에서 일괄 조치도 가능합니다. 미리보기는 항상 마스킹된 형태로만 보여줘, 검출 결과를 확인하는 과정에서 오히려 개인정보가 화면에 노출되는 것을 방지합니다. 이미지 검출 항목은 OCR로 찾은 위치를 표시합니다.

## 화면 6. 조치 확인 (특히 완전삭제 시)

비가역적 조치인 완전삭제 전에는 반드시 확인 단계를 둡니다. 실수로 인한 데이터 손실을 막는 안전장치입니다.

```
┌─────────────────────────────────────────────┐
│  ⚠ 완전삭제 확인                              │
│                                               │
│  선택한 3개 파일을 복구 불가능하게             │
│  영구 삭제합니다. 이 작업은 되돌릴 수 없습니다.│
│                                               │
│  · 고객명단.xlsx                              │
│  · config.yml                                 │
│  · 시안_신분증.png                            │
│                                               │
│  💡 권장: 먼저 [격리]로 보관 후 검토하세요.    │
│                                               │
│  확인을 위해 '삭제'를 입력하세요: [______]     │
│                                               │
│            [ 취소 ]   [ 격리로 변경 ]  [ 삭제 ]│
└─────────────────────────────────────────────┘
```

삭제 의도를 한 번 더 확인하기 위해 "삭제"를 직접 입력하게 하고, 더 안전한 대안인 격리로 전환할 수 있는 버튼을 함께 둡니다. 마스킹과 격리는 복원 가능하므로 이런 강한 확인 없이 진행합니다.

## 화면 7. 완료 및 리포트

조치가 끝나면 결과를 요약하고, 진단 리포트(PDF)를 발급합니다. 이 리포트는 발주처 보안 감사 증빙으로 쓰입니다.

```
┌─────────────────────────────────────────────┐
│  ✅ 점검 완료                                 │
│                                               │
│  처리 결과                                     │
│   마스킹  5건    격리  4건    삭제  3건         │
│   PC 위험 등급:  주의 🟡  →  안전 🟢            │
│                                               │
│  ┌─────────────────────────────────────┐    │
│  │  📄 점검 진단서 (PDF) 발급            │    │
│  │  발주처 보안 감사·법규 대응 증빙용    │    │
│  │              [ 리포트 저장 ]          │    │
│  └─────────────────────────────────────┘    │
│                                               │
│              [ 홈으로 ]   [ 격리함 보기 ]      │
└─────────────────────────────────────────────┘
```

위험 등급이 개선된 모습을 시각적으로 보여줘 사용자가 성취감을 느끼게 합니다.

## 보조 화면들

**격리함**은 격리된 파일을 목록으로 보여주며, 각 파일을 원위치로 복원하거나 영구삭제할 수 있습니다. 격리 일시와 원래 경로, 검출 사유를 함께 표시합니다.

**점검 이력(감사 로그)**은 언제 무엇을 스캔하고 어떤 조치를 했는지 시간순으로 기록해, 필터(기간·프로젝트·조치 유형)로 조회하고 내보내기(CSV/PDF)할 수 있습니다. 이 화면이 컴플라이언스 증빙의 근거가 됩니다.

**설정**은 직무 변경, 정기 자동 스캔 예약(주기·요일·시간), OCR 처리 방식(로컬 기본/외부 API 동의), 격리 폴더 위치, 제외할 폴더(화이트리스트) 등을 관리합니다. OCR 외부 API는 이미지가 PC를 벗어나므로 기본 비활성이며 명시적 동의 토글로만 켜집니다.

## 와이어프레임 설계의 핵심 원칙

전체 설계를 관통하는 원칙은 네 가지입니다.

1. 대시보드에서 "지금 점검하기"가 가장 크고 명확하게 단일 행동을 유도합니다.
2. 위험도를 색과 아이콘으로 직관화해 비전문가도 즉시 이해하게 합니다.
3. 미리보기와 결과는 항상 마스킹된 형태로만 노출해 점검 과정 자체가 새로운 유출 경로가 되지 않게 합니다.
4. 비가역적 삭제에는 반드시 확인 게이트를 두고 더 안전한 격리를 권장합니다.

이 와이어프레임은 글과 텍스트 목업으로 화면 구조·동선·구성요소를 모두 담고 있어, 디자이너가 바로 시각 디자인으로 옮기거나 개발자가 화면을 구현할 수 있는 수준입니다.
