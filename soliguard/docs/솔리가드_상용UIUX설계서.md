# 솔리가드(SoliGuard) 상용 수준 UI/UX 설계서

와이어프레임을 실제 출시 가능한 상용 제품 수준으로 끌어올린 설계서입니다. 디자인 시스템, UI/UX 고도화, 설치형 생애주기 전체 화면을 포함합니다.

---

## 1. 디자인 철학 및 원칙

상용 보안 도구의 디자인은 **신뢰감, 명료함, 통제감** 세 가지를 줘야 합니다. 사용자는 자신의 PC에서 민감한 데이터를 다루는 도구를 쓰는 만큼, 화면이 불안을 키우지 않고 차분하게 상황을 통제하고 있다는 느낌을 줘야 합니다.

이를 위한 다섯 가지 원칙을 둡니다.

1. **한 화면 한 목적** — 각 화면이 단 하나의 핵심 행동에 집중하게 합니다.
2. **진행 가시성** — 시간이 걸리는 스캔·OCR에서 사용자가 무슨 일이 일어나는지 항상 알게 합니다.
3. **위험의 시각적 위계** — 위험도를 색·아이콘·크기로 즉시 구분하게 합니다.
4. **되돌릴 수 있음** — 격리·마스킹은 복원 가능하게 하고 비가역 동작에는 강한 확인을 둡니다.
5. **신뢰의 가시화** — "로컬 처리"라는 핵심 약속을 화면 곳곳에서 반복 노출합니다.

---

## 2. 디자인 시스템 고도화

상용 수준에서는 색·타이포·간격을 토큰(token) 단위로 체계화해 일관성을 보장합니다.

**컬러**는 라이트/다크 두 테마를 지원합니다. 라이트 테마는 앞서 정의한 SoliBlue(`#1E40AF`) 계열을 메인으로, 다크 테마는 배경을 `#0F172A`(Slate900), 서피스를 `#1E293B`로 두고 강조색은 채도를 약간 높인 `#3B82F6`을 씁니다. 위험 신호색(빨강/노랑/초록)은 두 테마 모두에서 명도 대비를 WCAG AA 기준(4.5:1) 이상으로 맞춰 접근성을 확보합니다.

**타이포 스케일**은 12/14/16/18/22/28px의 6단계로 고정하고, 숫자·검출값은 등폭 폰트로 통일합니다. **간격**은 4px 그리드(4·8·12·16·24·32·48)를 엄격히 적용해 화면 전체가 정렬된 리듬을 갖게 합니다. **모서리 반경**은 버튼 8px, 카드 12px, 모달 16px로 위계를 둡니다.

**상태별 인터랙션**은 모든 인터랙티브 요소에 default/hover/pressed/focus/disabled 다섯 상태를 정의하고, 포커스 링(키보드 접근성)을 SkyBlue 2px로 일관되게 표시합니다. **모션**은 화면 전환 200ms ease-out, 카드 등장 150ms, 진행바·스피너는 부드러운 루프로 통일해 "기민하지만 산만하지 않은" 느낌을 줍니다.

## 3. UI/UX 고도화 포인트 (와이어프레임 대비)

상용 수준으로 끌어올리는 핵심 개선은 다음과 같습니다.

**대시보드의 정보 밀도와 위계 개선.** 단순 등급 표시를 넘어, 상단에 도넛 게이지(위험도별 비율 시각화)와 "지난 점검 대비 변화"(예: 위험 12건 → 4건, 개선 화살표)를 함께 보여줘 사용자가 추세를 읽게 합니다. "지금 점검하기" 버튼은 여전히 가장 크되, 그 아래 작은 카드로 "예약된 다음 점검: 6/9(월) 09:00"을 노출해 자동 점검이 돌고 있음을 안심시킵니다.

**스캔 진행의 체감 품질.** 단순 진행바 대신 단계 표시(파일 수집 → 검사 → 분석)와 함께, 실시간으로 발견 항목이 카운트업되는 애니메이션, 그리고 "예상 남은 시간"을 보여줍니다. OCR처럼 느린 단계는 별도 인디케이터로 "이미지 분석 중(시간이 걸릴 수 있어요)" 안내를 둬 멈춘 것으로 오해하지 않게 합니다.

**검출 결과의 탐색성.** 결과가 수백 건일 수 있으므로, 좌측에 필터 패널(위험도·정보유형·파일형식·폴더), 상단에 검색창, 그리고 정렬 옵션을 둡니다. 같은 파일의 여러 검출을 그룹으로 접고 펴는 아코디언으로 묶어 목록 피로를 줄입니다. 행을 클릭하면 우측 패널에 마스킹된 컨텍스트(검출값 주변 문장, 단 개인정보는 가림)를 보여줘 오탐 여부를 사용자가 판단하게 합니다. "이건 오탐이에요" 버튼으로 사용자가 화이트리스트에 추가하면 다음 스캔부터 제외되어 도구가 학습하는 느낌을 줍니다.

**조치의 안전한 UX.** 일괄 조치 전 "5개 파일, 12건이 처리됩니다" 요약을 보여주고, 완전삭제는 앞서 설계한 타이핑 확인에 더해 "삭제 전 자동 백업(격리본 보관)" 옵션을 기본 켜둬 실수를 구제합니다. 처리 중에는 행별로 진행 상태(처리 중→완료)가 인라인으로 바뀌고, 완료 후 토스트와 함께 실행취소(Undo) 스낵바를 잠시 노출합니다.

**빈 상태와 안심.** 검출 0건일 때 단순 "없음"이 아니라, 큰 초록 체크 일러스트와 "안전합니다 — 점검한 1,284개 파일에서 위험을 찾지 못했어요", 그리고 "다음 자동 점검 6/9" 안내로 긍정적 마무리를 줍니다.

## 4. 설치형 exe 프로세스에 맞는 전체 화면 흐름

설치형 프로그램은 "설치 → 최초 실행(초기 설정) → 일상 사용 → 백그라운드 자동 실행"이라는 생애주기가 있습니다. 각 단계에 맞는 화면을 설계합니다.

```
[설치 마법사]            (Inno Setup, exe 더블클릭)
  └ 환영 → 약관 → 설치경로 → 옵션(자동점검·바탕화면) → 설치진행 → 완료(실행)
        │
        ▼
[최초 실행 온보딩]        (앱 첫 구동 시 1회)
  └ 환영 → 직무 선택 → 스캔 폴더 지정 → 자동점검 주기 → OCR 동의 → 첫 점검 권유
        │
        ▼
[메인 앱]                (일상 사용)
  └ 대시보드 ⇄ 스캔설정 → 진행 → 결과 → 조치 → 완료/리포트
     보조: 격리함 · 이력 · 설정
        │
        ▼
[백그라운드 에이전트]     (작업 스케줄러가 주기적으로 구동)
  └ 무음 스캔 → 리포트 저장 → 트레이 알림(토스트)
```

### 4-1. 설치 마법사 화면 (Inno Setup 커스텀)

상용 인스톨러는 기본 회색 마법사가 아니라 브랜드가 입혀진 화면을 씁니다. 좌측에 SoliBlue 세로 배너(로고 + "SI 실무자를 위한 개인정보 점검 도구"), 우측에 단계별 내용을 둡니다.

환영 화면은 제품 한 줄 소개와 버전. 약관 화면은 라이선스와 함께 개인정보 처리 방침(로컬 처리 명시). 설치 경로 선택. 옵션 화면이 중요한데, 여기서 "바탕화면 아이콘 생성", "주 1회 자동 점검 사용(권장)" 체크박스를 두고, 자동 점검을 켜면 작업 스케줄러 등록이 이뤄짐을 안내합니다. 설치 진행바, 마지막 완료 화면에서 "지금 SoliGuard 실행" 체크와 함께 마칩니다. 관리자 권한이 필요하므로 UAC 안내를 자연스럽게 녹입니다.

### 4-2. 최초 실행 온보딩 (앱 첫 구동)

설치 직후 첫 실행에서만 나타나는 5단계 마법사로, 사용자를 첫 가치 경험(첫 점검)까지 빠르게 데려갑니다. 진행 점(progress dots)으로 단계를 표시합니다.

1. **환영** — 방패 로고와 "내 PC의 고객 데이터, 먼저 찾습니다", 핵심 약속 3개(로컬 처리·정확한 검출·법규 증빙)를 카드로 보여줍니다.
2. **직무 선택** — 개발자/디자이너/기획자/PM/전산사무 카드.
3. **스캔 폴더 지정** — 직무에 따라 추천 폴더(다운로드·바탕화면·프로젝트 폴더)가 미리 체크되어 있고 추가/제외할 수 있습니다.
4. **자동 점검 주기 설정** — 끔/매일/매주/매월 + 시각, 기본값은 "매주 월요일 09:00".
5. **OCR 동의** — "이미지 속 신분증·계약서도 검사할까요?"를 묻고 로컬 처리임을 명시합니다.

마지막에 "지금 첫 점검 시작" 버튼으로 곧장 가치를 경험하게 합니다.

### 4-3. 시스템 트레이 & 백그라운드 동작

상용 데스크톱 앱답게 시스템 트레이 상주를 지원합니다. 창을 닫아도 트레이로 최소화되며, 트레이 아이콘은 현재 위험 등급을 색(초록/노랑/빨강 점)으로 표시합니다. 트레이 우클릭 메뉴에 "지금 점검", "마지막 리포트 열기", "다음 예약: 6/9 09:00", "설정", "종료"를 둡니다. 자동 점검이 끝나면 윈도우 토스트로 "정기 점검 완료 — 위험 4건, 리포트 보기"를 띄우고, 클릭 시 결과 화면으로 진입합니다.

### 4-4. 설정 화면 고도화

설정은 탭으로 구조화합니다.

- **일반** — 직무·언어·테마(라이트/다크)
- **스캔** — 대상 폴더·제외 폴더·파일 형식·OCR 모드
- **자동 점검** — 주기·시각·자동 조치 수준
- **보안** — 격리 폴더 위치·삭제 정책·감사 로그 내보내기
- **정보** — 버전·업데이트 확인·라이선스

자동 조치 수준은 라디오로 "발견·리포트만(권장)" / "위험 파일 자동 격리"를 제공하고, 자동 완전삭제는 선택지 자체를 두지 않아 사고를 원천 차단합니다.

## 5. 마이크로인터랙션 & 디테일

상용 퀄리티는 디테일에서 갈립니다. 스캔 시작 시 버튼이 부드럽게 진행 상태로 모핑되고, 위험 등급 게이지는 숫자가 카운트업되며 채워지고, 조치 완료 시 해당 행이 살짝 페이드아웃되며 목록에서 정리됩니다. 호버 시 툴팁으로 검출 항목의 의미(예: "API키가 소스에 하드코딩되면 유출 위험이 큽니다")를 교육적으로 안내합니다. 로딩이 길어질 때는 보안 팁("프로젝트 종료 시 잔여 데이터를 정리하면 안전합니다")을 로테이션해 대기 시간을 가치 있게 만듭니다.

---

## 6. 접근성 & 반응형

키보드만으로 전체 흐름을 조작할 수 있게 탭 순서와 포커스 링을 정의하고, 스크린리더용 레이블을 모든 인터랙티브 요소에 부여합니다. 위험도는 색에만 의존하지 않고 아이콘·텍스트를 병기합니다. 창 크기는 1280×800 기본, 1024px 미만에서 사이드바가 아이콘 모드로 접히고, 결과 테이블은 우선순위 낮은 컬럼부터 숨겨 좁은 화면에서도 핵심(위험도·파일·조치)이 보이게 합니다. 다크 모드는 야간 작업이 많은 개발자를 배려한 기본 옵션으로 둡니다.

---

## 7. 디자인 산출물 체크리스트 (디자이너 핸드오프)

실제 제작을 위해 디자이너가 만들어야 할 산출물은 다음과 같습니다.

- 디자인 토큰 정의(색·타이포·간격·반경·그림자)
- 라이트/다크 두 테마의 컴포넌트 라이브러리(버튼·카드·태그·테이블·모달·토스트·트레이 메뉴)
- 설치 마법사 5화면
- 온보딩 5화면
- 메인 앱 9화면(대시보드·스캔설정·진행·결과·조치확인·완료·격리함·이력·설정)
- 빈 상태/에러 상태/로딩 상태 변형
- 마이크로인터랙션을 정의한 모션 스펙

이를 Figma에서 컴포넌트·베리언트로 구성하고 개발자에게 측정값과 함께 핸드오프합니다.

---

이 설계서는 와이어프레임을 실제 출시 가능한 상용 제품 수준으로 끌어올린 것으로, 디자인 시스템·UI/UX 고도화·설치형 생애주기 전체 화면을 모두 포함합니다. 디자이너는 이 문서를 기준으로 고해상도 시안 작업에, 개발자는 화면 구현과 트레이·인스톨러 연동에 바로 착수할 수 있습니다.
