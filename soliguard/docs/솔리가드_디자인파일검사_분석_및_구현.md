# 솔리가드(SoliGuard) 디자인 파일 검사 — 분석 & 구현

디자이너 타겟을 위한 디자인 파일(PSD · XD · Figma) 검사 기능을, **검사 가능성 분석**과 **정식 구현 코드** 두 부분으로 통합한 문서입니다.

---

# Part 1. 검사 가능성 분석 (PSD · XD · Figma)

디자이너 타겟을 포함하는 도구라면 반드시 고려해야 할 부분입니다. 결론부터 말하면 **가능하지만 포맷마다 난이도가 크게 다릅니다.** 각 포맷을 구체적으로 살펴봅니다.

## 1-1. 포맷별 검사 가능성

**PSD (Photoshop)** — 가능하며 비교적 용이합니다. PSD는 텍스트 레이어를 별도로 저장하므로, `psd-tools` 라이브러리로 텍스트 레이어의 문자열을 직접 추출해 기존 검출 엔진에 그대로 넣을 수 있습니다. 즉 "고객명단" 같은 텍스트 레이어에 박힌 주민번호는 OCR 없이 정확히 잡힙니다. 다만 텍스트가 아니라 이미지로 합쳐진(rasterized) 부분이나 스마트 오브젝트 안의 내용은 레이어를 이미지로 렌더링한 뒤 OCR을 돌려야 합니다.

**XD (Adobe XD)** — 가능합니다. `.xd` 파일은 실제로는 ZIP 컨테이너이고 내부에 JSON으로 아트보드·텍스트 데이터가 들어 있습니다. ZIP을 풀어 JSON을 파싱하면 텍스트 노드의 문자열을 추출할 수 있습니다. 다만 XD는 어도비가 신규 판매를 종료하고 지원을 축소하는 추세라(Figma에 통합 흡수) 우선순위를 낮게 두어도 무방합니다.

**Figma** — 가장 까다롭습니다. 이유는 Figma 디자인이 보통 로컬 파일이 아니라 클라우드(Figma 서버)에 저장되기 때문입니다. PC 스캔만으로는 디자인 본문에 접근할 수 없는 경우가 많습니다. 접근 경로는 두 가지입니다. 첫째, 로컬에 내려받은 `.fig` 파일이 있으면 검사 대상이 되지만, `.fig`는 비공개 바이너리 포맷이라 안정적 파싱이 어렵습니다. 둘째, 권장 방식은 Figma REST API를 쓰는 것입니다. 사용자가 발급한 액세스 토큰으로 파일의 텍스트 노드(`type: "TEXT"`의 `characters` 필드)를 읽어 검출 엔진에 넣을 수 있습니다.

### 설계상 중요한 모순

여기서 중요한 설계상 모순이 있습니다. 솔리가드의 핵심 약속은 "모든 데이터는 PC 안에서만 처리, 외부 미전송"입니다. 그런데 Figma API 검사는 본질적으로 클라우드에 있는 데이터를 가져오는 동작이라, 이 약속과 충돌합니다. 따라서 Figma 검사는 **기본 비활성**으로 두고, 사용자가 명시적으로 동의하고 토큰을 입력한 경우에만 별도 모드로 동작하게 설계하는 것이 옳습니다. 또한 디자이너가 검사하려는 것은 "내가 만든 디자인 안에 실고객 정보를 잘못 넣었는가"이므로, API로 가져온 텍스트를 로컬에서 검출만 하고 **가져온 원본은 즉시 폐기**하는 흐름이 적절합니다.

## 1-2. 정리

| 포맷 | 검사 가능 | 방식 | 난이도/우선순위 |
|---|---|---|---|
| PSD | 가능 | psd-tools로 텍스트 레이어 추출 + 래스터 부분 OCR | 낮음 / 높음 |
| XD | 가능 | ZIP 해제 → 내부 JSON 텍스트 추출 | 중간 / 낮음(XD 사양 축소) |
| Figma (.fig 로컬) | 제한적 | 비공개 바이너리, 파싱 불안정 | 높음 / 낮음 |
| Figma (클라우드) | 가능(별도 모드) | REST API + 토큰, 명시적 동의 필요 | 중간 / 중간 |
| 공통 한계 | — | 텍스트가 아닌 순수 이미지는 OCR 필수 | — |

## 1-3. 검출기 통합 방향 (스케치)

기존 `extractors.py`에 디자인 파일 추출기를 추가하면 됩니다. PSD는 텍스트 레이어를 합쳐 하나의 문자열로 만들어 `extract_text()`가 반환하고, XD는 JSON에서 텍스트 노드를 모읍니다. 초기 코드 스케치는 다음과 같습니다(정식 구현은 Part 2 참고).

```python
# extractors.py에 추가 (스케치)
SUPPORTED_DESIGN = {".psd", ".xd"}

def _extract_psd(path):
    """PSD 텍스트 레이어 추출 (psd-tools)"""
    from psd_tools import PSDImage
    psd = PSDImage.open(path)
    parts = []
    for layer in psd.descendants():
        if layer.kind == "type":              # 텍스트 레이어
            parts.append(layer.text)
        # 필요 시 래스터 레이어는 layer.composite() → OCR 폴백
    return "\n".join(p for p in parts if p)

def _extract_xd(path):
    """XD: ZIP 내부 JSON에서 텍스트 추출"""
    import zipfile, json, re
    texts = []
    with zipfile.ZipFile(path) as z:
        for name in z.namelist():
            if name.endswith(".json"):
                data = z.read(name).decode("utf-8", "replace")
                # 텍스트 노드의 "rawText"/"text" 필드 수집
                texts += re.findall(r'"rawText"\s*:\s*"([^"]*)"', data)
    return "\n".join(texts)
```

Figma 클라우드 검사는 파일 추출기와 분리된 별도 모듈(`figma_scan.py`)로 두고, 설정에서 토큰 입력 + 동의 토글이 켜졌을 때만 노출하는 것이 신뢰 설계상 안전합니다.

## 1-4. 권장 우선순위

디자이너가 주 타겟에 포함되므로 **PSD는 MVP 이후 2단계 차별화 기능**에 넣을 가치가 충분합니다. 텍스트 레이어 추출이 쉽고 효과가 확실하기 때문입니다. **XD는 사양 축소 추세라 후순위**, **Figma 클라우드 검사는 수요는 있으나 "로컬 처리" 원칙과의 조율이 필요하므로 별도 옵트인 기능으로 중간 우선순위**에 두는 것을 권합니다. 공통적으로, 디자인 파일에서 이미지로 합쳐진 개인정보(예: 시안에 붙인 신분증 사진)는 결국 OCR이 필요하므로, 이미 설계한 OCR 모듈과 연결해 두면 디자인 파일 검사의 사각지대를 메울 수 있습니다.

# Part 2. 정식 구현 코드

## 2-1. 디자인 파일 추출기 — `design_extractors.py`

기존 `extractors.py`에 통합되는 디자인 파일 처리 모듈입니다. PSD와 XD를 다루며, 텍스트 레이어를 우선 추출하고 이미지로 합쳐진 부분은 OCR로 폴백합니다. 통일된 인터페이스 `extract_text(path) -> str`를 유지합니다.

```python
# src/soliguard/design_extractors.py
"""디자인 파일(PSD/XD) 텍스트 추출 - 텍스트 레이어 우선, 래스터는 OCR 폴백.

설치 의존성:
    psd-tools>=1.9   (PSD)
    Pillow           (래스터 렌더링/OCR 입력)
    pytesseract      (OCR, 로컬)
XD는 표준 라이브러리(zipfile/json)만으로 처리.
"""

import json
import zipfile
import logging
from pathlib import Path

from soliguard.extractors import ExtractionError  # 공용 예외 재사용

log = logging.getLogger("soliguard.design")

SUPPORTED_DESIGN = {".psd", ".psb", ".xd"}

# 래스터 레이어 OCR 시 너무 큰 이미지로 인한 성능 저하 방지
_OCR_MAX_PIXELS = 4000 * 4000
_OCR_MAX_RASTER_LAYERS = 30   # PSD에서 OCR 돌릴 래스터 레이어 상한


def extract_design_text(path: Path, ocr_enabled: bool = True) -> str:
    """디자인 파일에서 텍스트 추출. 확장자별 분기."""
    path = Path(path)
    ext = path.suffix.lower()
    if ext in (".psd", ".psb"):
        return _extract_psd(path, ocr_enabled)
    if ext == ".xd":
        return _extract_xd(path)
    raise ExtractionError(f"지원하지 않는 디자인 형식: {ext}")


# ---------------------------------------------------------------------------
# PSD (Photoshop)
# ---------------------------------------------------------------------------

def _extract_psd(path: Path, ocr_enabled: bool) -> str:
    """PSD 텍스트 레이어 추출 + (옵션) 래스터 레이어 OCR.

    - type 레이어: layer.text 로 문자열 직접 추출 (정확, OCR 불필요)
    - 스마트오브젝트/래스터: 합성 후 OCR (시안에 붙인 신분증 이미지 등 대응)
    """
    try:
        from psd_tools import PSDImage
    except ImportError as e:
        raise ExtractionError("psd-tools 미설치로 PSD 검사 불가") from e

    try:
        psd = PSDImage.open(path)
    except Exception as e:
        raise ExtractionError(f"PSD 열기 실패: {e}") from e

    text_parts: list[str] = []
    raster_layers = []

    for layer in psd.descendants():
        kind = getattr(layer, "kind", None)
        if kind == "type":
            # 텍스트 레이어 - 가장 정확한 검출 경로
            try:
                t = layer.text
                if t:
                    text_parts.append(t)
            except Exception as ex:
                log.debug("텍스트 레이어 추출 실패(무시): %s", ex)
        elif ocr_enabled and kind in ("pixel", "smartobject"):
            # OCR 후보로 수집 (가시 레이어만)
            if getattr(layer, "is_visible", lambda: True)():
                raster_layers.append(layer)

    # 래스터/스마트오브젝트 OCR (상한 적용)
    if ocr_enabled and raster_layers:
        text_parts.extend(_ocr_psd_layers(raster_layers))

    result = "\n".join(p for p in text_parts if p and p.strip())
    return result


def _ocr_psd_layers(layers) -> list[str]:
    """PSD 래스터 레이어들을 이미지로 합성해 OCR."""
    try:
        import pytesseract
    except ImportError:
        log.info("pytesseract 미설치 - PSD 래스터 OCR 건너뜀")
        return []

    out = []
    for layer in layers[:_OCR_MAX_RASTER_LAYERS]:
        try:
            img = layer.composite()           # PIL.Image 반환
            if img is None:
                continue
            if img.width * img.height > _OCR_MAX_PIXELS:
                img.thumbnail((4000, 4000))   # 과대 이미지 축소
            txt = pytesseract.image_to_string(img, lang="kor+eng")
            if txt.strip():
                out.append(txt)
        except Exception as ex:
            log.debug("PSD 레이어 OCR 실패(무시): %s", ex)
    return out


# ---------------------------------------------------------------------------
# XD (Adobe XD) - ZIP 컨테이너 내부 JSON
# ---------------------------------------------------------------------------

# XD 내부 텍스트가 담기는 대표 키들 (버전에 따라 다름 → 여러 키 탐색)
_XD_TEXT_KEYS = ("rawText", "text", "value", "content")


def _extract_xd(path: Path) -> str:
    """XD(.xd)는 ZIP. 내부 artwork/*.json 등에서 텍스트 노드 추출."""
    try:
        zf = zipfile.ZipFile(path)
    except zipfile.BadZipFile as e:
        raise ExtractionError("유효한 XD(ZIP) 파일이 아님") from e

    texts: list[str] = []
    try:
        for name in zf.namelist():
            if not name.endswith(".json"):
                continue
            try:
                data = json.loads(zf.read(name).decode("utf-8", "replace"))
            except (json.JSONDecodeError, UnicodeDecodeError):
                continue
            _walk_xd_json(data, texts)
    finally:
        zf.close()

    return "\n".join(t for t in texts if t and t.strip())


def _walk_xd_json(node, acc: list[str]):
    """JSON 트리를 재귀 순회하며 텍스트 노드의 문자열 수집.

    XD는 텍스트를 보통 {"type":"text", "text":{"rawText":"..."}} 형태로 보관하나
    버전·내보내기 방식에 따라 구조가 달라 후보 키를 폭넓게 탐색한다.
    """
    if isinstance(node, dict):
        # 텍스트로 보이는 노드 처리
        if node.get("type") in ("text", "Text") or "rawText" in node:
            blob = node.get("text", node)
            if isinstance(blob, dict):
                for k in _XD_TEXT_KEYS:
                    v = blob.get(k)
                    if isinstance(v, str):
                        acc.append(v)
            elif isinstance(blob, str):
                acc.append(blob)
        # 그 외에도 직접 들고 있는 텍스트 키 수집
        for k in _XD_TEXT_KEYS:
            v = node.get(k)
            if isinstance(v, str):
                acc.append(v)
        # 자식 순회
        for v in node.values():
            if isinstance(v, (dict, list)):
                _walk_xd_json(v, acc)
    elif isinstance(node, list):
        for item in node:
            _walk_xd_json(item, acc)
```

이 모듈을 기존 `extractors.py`의 분기에 연결합니다.


추출기의 핵심 설계는 세 가지입니다. PSD는 텍스트 레이어를 OCR 없이 정확히 뽑아 "디자인 텍스트에 박힌 주민번호"를 잡고, 시안에 이미지로 붙인 신분증은 래스터 레이어 OCR로 폴백해 사각지대를 메웁니다. XD는 내부 JSON 구조가 버전마다 달라 후보 키를 폭넓게 재귀 탐색해 안정성을 확보했습니다. OCR은 성능 보호를 위해 이미지 크기·레이어 수에 상한을 둡니다.

## 2-2. Figma 옵트인 검사 모듈 — `figma_scan.py`

Figma는 클라우드 저장이 기본이라 "로컬 처리" 원칙과 충돌합니다. 따라서 기본 비활성, 명시적 동의·토큰 입력 시에만 동작, 가져온 데이터는 검출 직후 폐기하는 옵트인 모듈로 설계합니다.

```python
# src/soliguard/figma_scan.py
"""Figma 클라우드 디자인 검사 모듈 (옵트인 전용).

⚠ 신뢰 설계 원칙:
  - 솔리가드의 기본 약속은 '모든 데이터 로컬 처리, 외부 미전송'.
  - Figma 검사는 본질적으로 클라우드 데이터를 가져오므로 이 약속과 충돌한다.
  - 따라서 기본 비활성. 사용자가 (1)명시적 동의 + (2)토큰 입력 한 경우에만 동작.
  - 가져온 텍스트는 메모리에서 검출에만 사용하고 디스크에 저장하지 않으며,
    검출 종료 즉시 폐기한다. 이미지·원본 디자인은 내려받지 않는다(텍스트 노드만).
  - 감사 로그에는 'Figma 파일 검사함(파일키, 시각)'만 기록하고 내용은 남기지 않는다.

설치 의존성: requests
"""

import logging
from dataclasses import dataclass

from soliguard.scanner import scan_text
from soliguard.detectors import RULES, Finding

log = logging.getLogger("soliguard.figma")

FIGMA_API_BASE = "https://api.figma.com/v1"


class FigmaConsentError(Exception):
    """동의/토큰 없이 호출 시 - 옵트인 위반 차단"""


class FigmaApiError(Exception):
    """API 통신 실패"""


@dataclass
class FigmaScanResult:
    file_key: str
    file_name: str
    findings: list[Finding]
    text_node_count: int


def scan_figma_file(
    file_key: str,
    access_token: str,
    user_consented: bool,
) -> FigmaScanResult:
    """Figma 파일의 텍스트 노드를 가져와 로컬 검출 엔진으로 검사.

    Args:
        file_key: Figma 파일 URL의 키 (figma.com/file/<KEY>/...)
        access_token: 사용자가 발급한 개인 액세스 토큰
        user_consented: 외부 데이터 가져오기 명시적 동의 여부

    Returns:
        FigmaScanResult (검출 결과). 가져온 원본 텍스트는 반환하지 않음.
    """
    # ── 옵트인 가드: 두 조건 모두 충족해야 진행 ──
    if not user_consented:
        raise FigmaConsentError(
            "Figma 검사는 클라우드 데이터를 가져옵니다. 명시적 동의가 필요합니다."
        )
    if not access_token:
        raise FigmaConsentError("Figma 액세스 토큰이 필요합니다.")

    import requests

    headers = {"X-Figma-Token": access_token}
    try:
        resp = requests.get(
            f"{FIGMA_API_BASE}/files/{file_key}",
            headers=headers,
            timeout=30,
        )
    except requests.RequestException as e:
        raise FigmaApiError(f"Figma API 연결 실패: {e}") from e

    if resp.status_code == 403:
        raise FigmaApiError("토큰이 유효하지 않거나 파일 접근 권한이 없습니다.")
    if resp.status_code == 404:
        raise FigmaApiError("파일을 찾을 수 없습니다. file_key를 확인하세요.")
    if resp.status_code != 200:
        raise FigmaApiError(f"Figma API 오류: HTTP {resp.status_code}")

    doc = resp.json()
    file_name = doc.get("name", file_key)

    # ── 텍스트 노드만 수집 (이미지·디자인 원본은 가져오지 않음) ──
    texts: list[str] = []
    _collect_text_nodes(doc.get("document", {}), texts)
    node_count = len(texts)

    # ── 로컬 검출 엔진으로 검사 (가져온 텍스트는 메모리에만 존재) ──
    combined = "\n".join(texts)
    findings = scan_text(combined, RULES)

    # ── 원본 텍스트 즉시 폐기 ──
    texts.clear()
    combined = ""  # 참조 해제

    # ── 감사 로그: 내용 없이 메타만 기록 ──
    _audit_figma(file_key, file_name, len(findings), node_count)

    return FigmaScanResult(
        file_key=file_key,
        file_name=file_name,
        findings=findings,
        text_node_count=node_count,
    )


def _collect_text_nodes(node: dict, acc: list[str]):
    """Figma 문서 트리를 순회하며 TEXT 노드의 characters만 수집."""
    if not isinstance(node, dict):
        return
    if node.get("type") == "TEXT":
        chars = node.get("characters")
        if isinstance(chars, str) and chars.strip():
            acc.append(chars)
    for child in node.get("children", []):
        _collect_text_nodes(child, acc)


def _audit_figma(file_key: str, file_name: str, finding_count: int, node_count: int):
    """검출 내용은 기록하지 않고 검사 사실만 남긴다."""
    from soliguard.actions import write_audit
    write_audit(
        action="figma_scan",
        path=f"figma://{file_key}",
        result="success",
        extra={"file_name": file_name,
               "findings": finding_count,
               "text_nodes": node_count},
    )


def parse_file_key(figma_url: str) -> str:
    """Figma URL에서 파일 키 추출.
    예) https://www.figma.com/file/AbC123/제목  ->  AbC123
        https://www.figma.com/design/AbC123/제목 -> AbC123
    """
    import re
    m = re.search(r"figma\.com/(?:file|design)/([A-Za-z0-9]+)", figma_url)
    if not m:
        raise ValueError("올바른 Figma 파일 URL이 아닙니다.")
    return m.group(1)
```

### 옵트인 흐름 (설정·UI 연동)

설정·UI 연동에서 지켜야 할 옵트인 흐름은 다음과 같습니다. 설정 화면에 "Figma 클라우드 검사(고급)" 섹션을 두되 기본 꺼짐 상태로 둡니다. 사용자가 켜면 안내 문구("이 기능은 Figma 서버에서 디자인의 텍스트를 가져옵니다. 가져온 내용은 검사 후 즉시 폐기되며 저장되지 않습니다")와 함께 동의 체크박스, 토큰 입력란, 검사할 파일 URL 입력란을 노출합니다. 동의와 토큰이 모두 있어야 `user_consented=True`로 호출됩니다.

```python
# UI 연동 예시 (gui/settings 측)
def run_figma_scan(url: str, token: str, consent: bool):
    from soliguard.figma_scan import scan_figma_file, parse_file_key, FigmaConsentError, FigmaApiError
    try:
        key = parse_file_key(url)
        result = scan_figma_file(key, token, user_consented=consent)
    except FigmaConsentError as e:
        return show_error("동의 필요", str(e))
    except (FigmaApiError, ValueError) as e:
        return show_error("Figma 검사 실패", str(e))
    # result.findings 를 일반 검출 결과 화면에 병합 표시 (마스킹된 값만)
    return result
```

이 모듈의 핵심 설계는 네 가지입니다. 동의와 토큰 두 조건을 모두 충족해야만 동작하는 **옵트인 가드**, 이미지·디자인 원본은 가져오지 않고 TEXT 노드의 문자열만 수집하는 **최소 수집**, 검출 직후 가져온 텍스트를 메모리에서 폐기하는 **비저장 원칙**, 그리고 감사 로그에 검출 내용 없이 "검사함" 사실만 남기는 **기록 분리**입니다. 이렇게 하면 디자이너의 실수(디자인에 실고객 정보 삽입)를 잡으면서도 "로컬 처리" 신뢰 원칙과의 충돌을 최소화합니다.
